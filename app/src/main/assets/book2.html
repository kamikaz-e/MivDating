Основы разработки на Android

Глава 1: Архитектура Android

Android - это операционная система на базе Linux, разработанная специально для мобильных устройств. Архитектура Android состоит из нескольких слоев: ядро Linux, библиотеки, Android Runtime, фреймворк приложений и приложения.

Ядро Linux обеспечивает основные системные сервисы, такие как управление памятью, процессами, сетевым стеком и драйверами устройств. Поверх ядра располагаются системные библиотеки, написанные на C/C++, которые предоставляют функциональность для работы с графикой, базами данных, медиа и другими компонентами.

Глава 2: Jetpack Compose

Jetpack Compose - это современный декларативный UI-фреймворк для Android. Вместо XML-разметки разработчики описывают интерфейс с помощью Kotlin-функций, которые называются composables.

Основные преимущества Compose включают: меньше boilerplate кода, реактивное программирование, лучшую производительность и простоту тестирования. Compose автоматически обновляет UI при изменении состояния, что упрощает разработку динамических интерфейсов.

State management в Compose основан на концепции однонаправленного потока данных. Состояние хранится в ViewModel, а composable функции отображают это состояние. При изменении состояния Compose автоматически перерисовывает только измененные части UI.

Глава 3: Kotlin для Android

Kotlin - это официальный язык программирования для разработки Android-приложений. Он полностью совместим с Java, но предоставляет множество современных возможностей, которые делают код более безопасным и выразительным.

Null-safety - одна из ключевых особенностей Kotlin. Система типов различает nullable и non-nullable типы, что позволяет избежать NullPointerException на этапе компиляции. Корутины упрощают асинхронное программирование, делая код более читаемым и поддерживаемым.

Глава 4: Работа с сетью

Для работы с сетевыми запросами в Android используются различные библиотеки. Retrofit - популярная библиотека для работы с REST API. Она автоматически преобразует JSON-ответы в Kotlin объекты и поддерживает корутины для асинхронных запросов.

Ktor Client - это мультиплатформенная HTTP-клиентская библиотека от JetBrains. Она написана на Kotlin и идеально интегрируется с корутинами. Ktor поддерживает различные движки для разных платформ и предоставляет гибкую систему плагинов для логирования, сериализации и аутентификации.

Глава 5: Хранение данных

Android предоставляет несколько способов хранения данных: SharedPreferences для простых key-value пар, Room для работы с SQLite базами данных, и DataStore для асинхронного хранения настроек с поддержкой корутин.

Room - это ORM библиотека, которая предоставляет абстракцию над SQLite. Она использует аннотации для определения таблиц и запросов, проверяет SQL-запросы на этапе компиляции и автоматически генерирует код для работы с базой данных.

DataStore - современная замена SharedPreferences. Она основана на Kotlin корутинах и Flow, предоставляет типобезопасный API и поддерживает миграцию данных. Существует две версии: Preferences DataStore для простых типов данных и Proto DataStore для работы с Protocol Buffers.
